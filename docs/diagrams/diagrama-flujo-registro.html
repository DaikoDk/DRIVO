Flujo de Registro de Cliente

Validaciones • JWT Auto-login • Redirección al Portal

sequenceDiagram
    autonumber
    actor Cliente
    participant Frontend
    participant AuthController
    participant ClienteService
    participant BD

    Cliente->>Frontend: Completa formulario (datos + licencia + contraseña)
    Frontend->>AuthController: POST /api/auth/register (público)
    AuthController->>ClienteService: registrarCliente(dto)

    ClienteService->>BD: SELECT COUNT(*) FROM tb_cliente WHERE dni = ?
    BD-->>ClienteService: Resultado

    alt DNI ya registrado
        ClienteService-->>AuthController: 409 Conflict (DNI duplicado)
        AuthController-->>Frontend: 409
        Frontend-->>Cliente: ❌ DNI ya existe
    else DNI disponible
        ClienteService->>BD: SELECT COUNT(*) FROM tb_usuario WHERE Correo = ?
        BD-->>ClienteService: Resultado

        alt Email ya registrado
            ClienteService-->>AuthController: 409 Conflict (Email duplicado)
            AuthController-->>Frontend: 409
            Frontend-->>Cliente: ❌ Email ya existe
        else Email disponible
            ClienteService->>ClienteService: Valida contraseña (mínimo 6)

            alt Contraseña muy corta
                ClienteService-->>AuthController: 400 Bad Request
                Frontend-->>Cliente: ❌ Contraseña muy corta
            else Contraseña válida
                ClienteService->>BD: INSERT usuario (BCrypt) + licencia + cliente
                ClienteService->>AuthController: generarToken(email, CLIENTE)
                AuthController->>AuthController: Firma JWT (expiración 24h)
                AuthController-->>Frontend: 201 Created + { token, rol: "CLIENTE" }
                Frontend->>Frontend: Guarda token, redirige al portal
                Frontend-->>Cliente: ✅ Sesión iniciada automáticamente
            end
        end
    end
    Note over ClienteService,BD: 🔐 JWT auto-login: sin paso extra de login tras registrarse.
