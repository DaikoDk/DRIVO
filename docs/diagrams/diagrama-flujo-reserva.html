Flujo Completo de Reserva

Hold temporal (24h) • 8 estados unificados • Cálculo automático de costos

sequenceDiagram
    autonumber
    actor Cliente
    actor Administrador
    participant Frontend
    participant ReservaService
    participant AutoService
    participant BD

    Cliente->>Frontend: Selecciona auto y fechas
    Frontend->>ReservaService: POST /api/reservas (Bearer JWT)
    ReservaService->>AutoService: verificarDisponibilidad()
    AutoService->>BD: SELECT solapamiento de fechas
    BD-->>AutoService: Sin conflictos
    AutoService-->>ReservaService: Disponible
    ReservaService->>BD: INSERT tb_reserva (PENDIENTE)
    ReservaService->>AutoService: cambiarEstado(auto, RESERVADO)
    AutoService->>BD: UPDATE tb_auto SET estado='RESERVADO'
    Note over ReservaService,AutoService: 🔒 HOLD 24h: Si no confirma, scheduler libera el auto solo.

    Administrador->>Frontend: Inicia check-in (km inicial)
    Frontend->>ReservaService: PUT /api/reservas/{id}/iniciar
    ReservaService->>BD: UPDATE estado=EN_CURSO, kmInicio

    Administrador->>Frontend: Finaliza check-out (km final, daños)
    Frontend->>ReservaService: PUT /api/reservas/{id}/finalizar
    ReservaService->>ReservaService: Calcula total = subtotal + mora + daños
    ReservaService->>BD: UPDATE estado=ENTREGADO, kmFin, costos
    ReservaService->>AutoService: cambiarEstado(auto, DISPONIBLE)
    AutoService->>BD: UPDATE tb_auto SET estado='DISPONIBLE'

    Administrador->>Frontend: Registra pago
    Frontend->>ReservaService: POST /api/pagos
    ReservaService->>BD: INSERT pago + UPDATE reserva → FINALIZADA
    Note over ReservaService,BD: 🗂️ Una tabla tb_reserva con 8 estados unifica reserva y alquiler.
